# WIMS — نظام إدارة المخازن والمخزون (دليل التشغيل)

هذه حزمة تشغيل ذاتية لجهاز Windows واحد. لا تحتاج تنصيب .NET.

## المتطلبات
- **Windows 10/11** (أو Windows Server 2019+).
- **SQL Server Express** مُنصَّب (مجاني). حمّله من موقع Microsoft إن لم يكن موجوداً، واسمُه الافتراضي غالباً `.\SQLEXPRESS`.

## التنصيب (مرة واحدة)
1. فُكّ ضغط مجلد الحزمة في مكان **قابل للكتابة** (مثل سطح المكتب أو مجلد المستندات) — لا تضعه في `Program Files`.
2. اضغط بزر الفأرة الأيمن على **`setup.ps1`** ثم **Run with PowerShell** (أو من PowerShell: `./setup.ps1`).
   - أدخِل اسم SQL instance (أو اترك الافتراضي `.\SQLEXPRESS`).
   - سيولّد النظام مفتاح أمان تلقائياً.
   - أدخِل كلمة مرور قوية لحساب المدير `admin` (8 أحرف على الأقل، تتضمّن حرفاً كبيراً وصغيراً ورقماً ورمزاً).

> إذا ظهر خطأ «تشغيل السكربتات معطّل»، افتح PowerShell وشغّل:
> `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` ثم أعد تشغيل `setup.ps1`.

## التشغيل اليومي
- شغّل **`Start-WIMS.bat`** بالنقر المزدوج.
- سيفتح المتصفح تلقائياً على **http://localhost:5000**.
- سجّل الدخول: المستخدم **`admin`** وكلمة المرور التي ضبطتها.
- **أول خطوة:** من قائمة المستخدم أعلى الشاشة → «تغيير كلمة المرور» إن رغبت، ثم من «الإدارة والصلاحيات» أنشئ الأدوار والمستخدمين.
- لإيقاف النظام: أغلِق نافذة **WIMS** السوداء.

> عند أول تشغيل تُنشأ قاعدة البيانات `WIMS` تلقائياً على SQL Express وتُهيَّأ البيانات الأساسية والأدوار.

## إدارة المستخدمين والصلاحيات
من شاشة **«الإدارة والصلاحيات»** (متاحة لمدير النظام):
- **المستخدمون:** إضافة موظف، إسناد أدوار، تفعيل/تعطيل، إعادة تعيين كلمة المرور.
- **الأدوار:** إنشاء دور وتحديد صلاحياته من مصفوفة الصلاحيات. أدوار جاهزة: *أمين مخزن، معتمِد، مدير مالي، مدقّق*.

## الوصول من أجهزة أخرى على الشبكة (اختياري)
افتراضياً النظام محلي فقط (localhost). لإتاحته على الشبكة المحلية:
1. عدّل `Start-WIMS.bat`: غيّر `ASPNETCORE_URLS` إلى `http://+:5000`.
2. اسمح للمنفذ 5000 في جدار الحماية.
3. **يُنصح بشدّة بتفعيل HTTPS** في هذه الحالة (عيّن `Security:RequireHttps=true` واضبط شهادة)، لأن كلمات المرور والرموز تُرسل عبر الشبكة.

## النسخ الاحتياطي
البيانات في قاعدة `WIMS` على SQL Express. خذ نسخة احتياطية دورية عبر SQL Server Management Studio أو مهمة SQL Agent.

## استكشاف الأعطال
- **الصفحة لا تفتح:** انتظر ثوانٍ ثم حدّث المتصفح؛ تأكد أن نافذة WIMS لم تُغلق وأن SQL Express يعمل.
- **فشل الإقلاع بخطأ مفتاح JWT:** أعد تشغيل `setup.ps1` (يجب أن يكون المفتاح مضبوطاً).
- **سجلّات التشغيل:** داخل `app\logs\`.
