# سجل التغييرات (Changelog)

هذا الملف يوثّق كل التغييرات الملحوظة في نظام **WIMS** (نظام إدارة المخازن والمخزون — وزارة العدل)، إصداراً بإصدار.

يتّبع تنسيق [Keep a Changelog](https://keepachangelog.com/ar/1.1.0/) وترقيم [Semantic Versioning](https://semver.org/lang/ar/) (MAJOR.MINOR.PATCH).

## كيف يعمل هذا النظام

- كل تغيير يُسجَّل أولاً تحت قسم **[غير مُصدَر]** أثناء العمل عليه.
- عند إعلانك رقم إصدار (مثال: "خلّيها 1.1.0")، يُنقل محتوى **[غير مُصدَر]** إلى قسم جديد بعنوان الإصدار والتاريخ، ويُنشأ **git tag** بنفس الرقم، ويُحدَّث رقم الإصدار في `Directory.Build.props` و`wims-client/package.json`.
- بداية من أول إصدار موسوم، تُستخرج تفاصيل كل إصدار تالٍ تلقائياً بمقارنة (`git diff`) الوسم السابق بالحالة الحالية — لضمان دقة السجل.

## [0.1.0] — نقطة الأساس (قبل بدء نظام الإصدارات) — 2026-07-04

هذا أول إصدار مُوثَّق، ويلخّص كل ما بُني في المشروع قبل تفعيل نظام الإصدارات والـ Git — كنقطة انطلاق مرجعية فقط، وليس نتيجة مقارنة Git دقيقة (لم يكن المستودع تحت Git حينها).

### أُضيف
- **الأساس التشغيلي (المراحل 0-6):** كتالوج الأصناف، سندات الحركة (إضافة/صرف/تحويل/تسوية/مرتجع)، محرّك اعتماد بخطوات موجّهة حسب الدور، استيراد Excel (أصناف/أرصدة افتتاحية)، الجرد الفعلي ومحاضره، تنبيهات المخزون، التقارير (عرض + تصدير PDF/Excel)، لوحة مؤشرات رئيسية.
- **نظام صلاحيات (RBAC) تشغيلي كامل:** CQRS لإدارة المستخدمين/الأدوار/الصلاحيات، واجهة إدارة كاملة (تبويبَي مستخدمين وأدوار + مصفوفة صلاحيات)، تغيير كلمة المرور الذاتي، توجيه المصادقة الديناميكي (`PermissionPolicyProvider`)، أدوار دقيقة مبذورة (مدير النظام/أمين مخزن/معتمِد/مدير مالي/مدقّق) بدل دور واحد شامل، توجيه الموافقات حسب الدور الفعلي للمستخدم.
- **إخفاء عناصر الواجهة حسب الصلاحية:** توجيه بنيوي `*appHasPermission` + حماية مسارات.
- **العُهد:** كشف عهدة موظف، براءة الذمة، وميزة **إرجاع عهدة فردية للمخزن** (جديدة بالكامل) — تُنشئ سند مرتجع مُعتمَد تلقائياً وتُعيد الرصيد فعلياً.
- **تغليف وتسليم ذاتي (SelfHost):** نشر self-contained single-file لجهاز Windows واحد مع SQL Server Express، استضافة الواجهة من نفس origin الـ API، خطوط عربية مستضافة ذاتياً (offline)، سكربتات `build-package.ps1` / `setup.ps1` / `Start-WIMS.bat`، دليل مستخدم عربي (`README-User-AR.md`).
- **حزمة نشر MonsterASP.NET:** `publish-monsterasp.ps1` (نشر framework-dependent win-x86)، `appsettings.Production.json` مهيّأ لصيغة اتصال MonsterASP، ودليل نشر خطوة بخطوة (`DEPLOY-MonsterASP-AR.md`).
- **مواد تدريب المستخدم:** دليل استخدام مصوّر تفاعلي (HTML) + نسخة Word للطباعة، سكريبت تسجيل فيديو تعليمي بالعامية المصرية، واستوديو فيديو تفاعلي (HTML) يُنتج ملف فيديو حقيقي (MP4/WebM) تلقائياً بصوت آلي أو بصوت المستخدم.
- **قرار سياسة معتمَد:** إلغاء قيد "لا يجوز اعتماد سند أنشأته بنفسك" لكل أنواع السندات بلا استثناء (تغيير مقصود، موثَّق في `plan.md`).

### أُصلح
- **`AuditBehavior`:** كان يسجّل ويحفظ سجل التدقيق حتى عند فشل الأمر فعلياً؛ الآن يُسجَّل فقط عند نجاح حقيقي.
- **`CustodyProvisioningService`:** إضافة بند عهدة عبر `parent.Items.Add(...)` كانت تُصنَّف خطأً كـ`Modified` بدل `Added` (فخّ EF Core شائع) مسبّبةً `DbUpdateConcurrencyException`؛ استُبدلت بإضافة صريحة عبر `db.CustodyItems.Add(...)`.
- **NG0600 (Angular):** توجيه `*appHasPermission` داخل `effect()` بلا `allowSignalWrites` كان يُسقط عرض بعض الأزرار (مثل زر الاعتماد) بصمت عند تعايشه مع `viewChild.required()` في نفس المكوّن.
- **`appsettings.Production.json`:** كان يشير إلى Serilog sink باسم `MSSqlServer` غير مثبَّت أصلاً كحزمة NuGet في المشروع — كان سيُفشل الإقلاع فوراً عند أي نشر فعلي بيئة Production؛ استُبدل بسجلّ ملفّي (File) متوافق مع الحزم المثبَّتة فعلياً.
- تعطّل عشوائي في `RateLimitingTests` بسبب تسمّم قاعدة بيانات الاختبار المشتركة عبر قفل Identity الحقيقي على مستخدم `admin`.

### تغييرات تقنية أخرى
- `.gitignore` مُعدّ مسبقاً لتتبّع `wwwroot/.gitkeep` مع تجاهل محتوى البناء.
- بدء استخدام Git لأول مرة لهذا المستودع (`git init`) لتفعيل تتبّع تاريخ دقيق للإصدارات القادمة.
